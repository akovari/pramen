# v0.1 release checklist (P2.2)

Use this before tagging `0.1.0` and pushing a GitHub Release. Every item
must be green; cloud-only legs are explicitly marked and do not block the
offline-first release.

## Binary and installers

- [ ] `dist plan` succeeds locally (`mise exec -- dist plan`)
- [ ] Release workflow dry-run on a PR shows the expected four tier-1
      artifacts (Linux x86_64/aarch64 musl, macOS aarch64, Windows x86_64)
      plus shell and PowerShell installers
- [ ] `pramen --version` reports `0.1.0` on each tier-1 target

## Fresh-machine quickstart

- [ ] `scripts/release-quickstart.sh` passes on a clean checkout with only
      `mise install` / `cargo build --release` (or a downloaded release
      binary) and a writable PostgreSQL — under the ten-minute bar
- [ ] CI quickstart job passes (debug build; release script is the local
      gate for the release profile)

## Documentation

- [ ] README lists known limitations honestly (see below)
- [ ] Examples run as documented (`examples/local-parquet-to-postgres.yaml`,
      `examples/local-tickets-ai-classify.yaml`,
      `examples/local-tickets-ai-classify-batch.yaml`)
- [ ] Docs site installation page updated for release binaries
- [ ] `docs/implementation-plan.md` snapshot reflects v0.1 ship state

## Quality gates (unchanged from every PR)

- [ ] `mise run ci` green on Linux, macOS, and Windows tier-1 matrix
- [ ] L2 Postgres job green
- [ ] Perf workflow green on designated benches (or no regression accepted
      via ADR)

## Known limitations (ship with these stated, not hidden)

- **Single-node, single-connection sink.** No horizontal scaling, no parallel
  COPY, no fan-out DAGs.
- **PostgreSQL only** for sinks; ADBC and Flight SQL are post-v1.
- **Linear pipelines only** — one source, ordered transforms, one sink.
- **Bedrock batch minimum job size.** The service enforces a minimum records
  per model invocation job (1,000 at the time of writing); smaller `execution:
  batch` runs fail at provider validation.
- **Live cloud acceptance not measured in CI.** Bedrock Converse online
  (S1.1), Bedrock batch crash/reconcile on real infrastructure (S2.1), the
  model quality-cost frontier (S2.2), and the 1M-record AWS run (P2.1) need
  credentials; PR gates are fully offline per ADR 0005.
- **WASM transforms are Phase 2.** The ABI is spike-validated (S1.4) but not
  wired into the runtime.
- **Review queue is CLI-only.** No web UI; low-confidence routing needs a
  provider confidence signal not yet available.

## Cloud acceptance (post-release, does not block tag)

- [ ] S1.1 live Bedrock Converse leg in `eu-central-1`
- [ ] S2.1 crash/reconcile numbers on real Bedrock batch
- [ ] S2.2 frontier table published as an ADR
- [ ] P2.1 1M-record S3 → Aurora acceptance run
